Overview

RECOBeaconRegion object defines a type of region that is based on the device’s proximity to a Bluetooth beacon. A beacon region looks for devices whose identifying information matches the information you provide. When that device comes in range, the region triggers the delivery of an appropriate notification.

You can monitor beacon regions in two ways. To receive notifications when a device enters or exits the vicinity of a beacon, use the startMonitoringForRegion: method of your RECOBeaconManager object. While a beacon is in range, you can also call the startRangingBeaconsInRegion: method to begin receiving notifications when the relative distance to the beacon changes.

A RECOBeaconRegion object contains an instance of CLRegion which can be obtained by getCLRegion method. For more details about CLRegion, please refer to here

initWithProximityUUID:identifier:

Initializes and returns a region object that targets a beacon with the specified proximity ID.

- (id)initWithProximityUUID:(NSUUID *)uuid identifier:(NSString *)identifier

Parameters

uuid

The unique ID of the beacons being targeted. If this value is nil, UUID will be set to the one provided by Perples Inc. by default.

identifier

A unique identifier to associate with the returned region object. You use this identifier to differentiate regions within your application. This value must not be nil.

Return Value

An initialized RECOBeaconRegion object.

Discussion

Initializes and returns a region object that targets a beacon with the specified proximity ID.

This method creates a region that results in the reporting of all beacons with the specified proximityUUID value. The major and minor values of the beacons are ignored.
